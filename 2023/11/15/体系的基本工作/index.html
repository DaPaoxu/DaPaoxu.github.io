体系的基本工作

1. 资料维护

（解放了就没多少活了，就解放了，可以去检查改善）

​ 从作业指引入手

2. 内审

管理体系内审要点：

审核：客观地获取审核证据并予以评价，以判断特定的活动、事件、状况、管理体系，或有关上述事项的信息是否符合 → 的一个系统化并 → 的验证过程

证据 → 管理体系运行的证据 → 记录

活动 → 过程

事件 → 判断、评价、分析

状况 → 异常

步骤

1.审核计划 确定大概时间 几月份

2 审核准备 内审实施计划：条款号

内审检查表 （内审的核心）： 条款 → 文件 → 检查内容点 → 提供证据 → 被审核方提供

3 审核实施 先和负责人开会 再和审核小组开会（培训使用内审检查表、开不符合项报告） 开出来报告后要负责人当场签字

​ 从部门分析哪个部门问题多，或者从条款分析哪些问题多

4 审核报告 做出来后一定要审核组长确认 公司经理确认签字

5 跟进

不符合项 ：建议项 一般项 严重项

解决有没有后就基本不会有严重不符合

3. 第三方审核

1-6：资料，所有记录必须有

7： 资料，年度培训计划，培训实施记录，文件收发受控记录

8：资料，近三个月，每一类提供8份（时间分配到近三个月，最好不要连着拿一个月的）

​ 研发记录提供近一年内的1-2份（支撑范围的记录）

4. 管理评审

建议11月内审，12月管理评审

审核所审的资料对于文件记录的要求 ：倒推一年内

第三方可以准备最近三个月的资料

输入：内审报告，各部门年度总结（应包含体系运行情况），各类重大事件（对管理体系有影响的）

输出：管理评审报告，改善项目，签到表，会议记录

年度总结会议+管理评审 xxx管理体系标准文件

二三方审核审的不是体系，体系只是一个资料的收集和接待部门，记录是各个部门给的

不要当老好人，谁配合我工作我就配合谁，爱憎分明

上年度管理评审会议决议事项的执行情况

5. 审核接待注意事项

管理型工作：

​ 主打的就是一个见缝插针

​ 分析 → 明确问题 → 管理改善方向 → 对应到相应的文件

填表的过程 → 拆分 → 按部门分配 → 回收整合

一般为改去年的内审和管理评审表
